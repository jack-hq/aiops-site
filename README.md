## 产品简介

AI Ops Agent 聚焦生产环境中的常见告警自动处理，形成从告警进入到恢复确认的闭环。

### 核心能力

- **自动诊断**
  接收 Prometheus / Alertmanager 告警后，自动分析服务状态、异常线索和上下文信息，快速定位常见故障原因。
- **自动修复**
  根据预定义场景与执行协议，自动生成并执行修复步骤，减少人工重复操作。
- **自动验证**
  修复完成后自动校验服务是否恢复，避免“动作执行了，但业务还没恢复”的问题。
- **过程可审计**
  保留诊断、计划、执行、验证全流程记录，方便复盘、交付和后续治理。

---

## 适用对象

适合以下类型的团队：

- 已使用 Prometheus / Alertmanager 的技术团队
- 已有 Kubernetes 环境，希望减少人工值班成本的团队
- 存在大量重复性故障处理动作的团队
- 希望先从 1 个高频告警场景开始验证自动修复价值的团队

---

## 典型场景

当前页面重点展示以下自动修复场景：

- Web 服务异常自动恢复（如 Nginx）
- 数据库服务异常自动恢复（如 MySQL）
- Kubernetes Pod CrashLoop 自动恢复
- 常见服务不可用告警自动处理

## 动态演示模块说明

页面中的“动态演示”模块用于模拟一次典型自动修复流程，当前通过前端本地状态驱动实现。
- 演示步骤
- 接收告警
- 自动分析
- 匹配场景
- 生成处理步骤
- 执行修复
- 验证恢复
- 实现方式

## 主要通过以下数据结构控制：
- demoSteps：演示步骤列表
- demoTimeline：每一步对应的状态数据与日志
- demoCurrentStep：当前步骤索引
- demoState：当前状态卡片展示内容
- demoLogs：处理过程日志
- demoPlaying：是否正在播放
- demoTimer：定时器实例

## 逻辑说明
点击“播放演示”后：
- 重置当前状态
- 启动定时器
- 按步骤推进演示
- 每推进一步，更新状态卡片和日志内容
- 最后停留在“告警已清除”状态
